# F-002 · > **Mechanisms 论文揭示的 object containerization、稀疏定位关键 attention heads、“先识别、后定位”的顺序

## Finding

> **Mechanisms 论文揭示的 object containerization、稀疏定位关键 attention heads、“先识别、后定位”的顺序机制以及空间位置重建机制，能否解释 FOCUS 的 attention loss 为什么有效？FOCUS 当前“跨所有层和头平均 attention，并监督 query image tokens 关注 support BBOX 文本 token”的设计是否真正命中了定位的因果计算通路？能否据此设计更直接、更稀疏、训练成本更低的 ICOL 监督方法？**

这个表述把讨论分成了三个层次：先解释 FOCUS，随后质疑 FOCUS 的具体实现，最后提出可验证的改进，而不是泛泛地说“mechanism 对 FOCUS 有启示”。

## 建议拆分的分析维度

### 1. FOCUS 的提升究竟来自普通 SFT，还是来自定位机制定向干预？

这是最应该先回答的问题。

FOCUS 的消融结果显示，普通 SFT 并不能解释性能提升。在 TAO 4-shot 上，冻结的 Qwen2-VL-7B 为 36.1，普通 SFT 降至 34.3，单独 GRPO 降至 32.1，SFT+GRPO 也只有 33.7；加入 attention loss 后才上升到 57.1，再加入 GRPO 达到 68.5。换言之，**关键并不是“训练了模型”，而是加入了特定的 attention 约束**。

Mechanisms 论文则发现，定位依赖少量具有因果作用的注意力头，并明确提出未来可以进行 targeted head fine-tuning 或 grounding-aware attention supervision。这为 FOCUS 的结果提供了一个更准确的解释：FOCUS 有效，可能不是因为泛化意义上的 SFT，而是因为它间接改变了定位相关的注意力计算。

建议将这一轴命名为：

> **FOCUS 的主要收益来自 SFT，还是来自机制对齐的 attention supervision？**

初步结论是：论文证据更支持后者。

---

### 2. FOCUS 监督的 attention 是否是实际的因果定位通路？

FOCUS 将所有层、所有注意力头的 attention matrix 直接平均，然后要求 query image token 对 support BBOX annotation token 的平均注意力高于非 BBOX token。这里的 BBOX token 是序列化坐标文本，如 `[xmin, ymin, xmax, ymax]`，并不是 support 图像中框内的视觉 token。

但 Mechanisms 的 CMA 结果表明：

* 绝大多数 attention head 对定位几乎没有因果贡献；
* 定位信息集中在很少的专门化 head；
* 不同架构的关键层位置不同；
* 部分 head 甚至具有负 Mediation Fraction，即其激活可能提供反证或干扰信息。

因此，FOCUS 将所有层和头平均后共同施加监督，可能存在两个问题：

1. 真正关键 head 的梯度被大量无关 head 稀释；
2. 无关甚至具有负贡献的 head 也被迫学习同一种 attention 模式。

不过，这目前只是由 Mechanisms 推导出的假设，不能说 FOCUS 的全局平均一定错误，因为 FOCUS 的消融已经证明该目标在性能上有效。

建议将该轴表述为：

> **FOCUS 的全层、全头平均 attention loss，是否命中了少量具有因果作用的 localization heads，还是仅制造了与性能相关但非因果的 attention pattern？**

最直接的验证方法是对 FOCUS 训练前后的 Qwen2-VL 做 attention knockout、CMA 和关键 head 累积消融，观察：

* FOCUS 是否增强了原有定位 head；
* 是否产生了新的 support-conditioned 定位 head；
* 去掉这些 head 后，FOCUS 的性能增益是否消失。

---

### 3. FOCUS 是否监督了错误的 token 类型？

FOCUS 声称强制模型利用视觉 support，但它的 attention loss 实际监督的是：

```text
query image visual tokens → support BBOX coordinate text tokens
```

它并未直接要求 query token 关注 support bbox 内部的视觉 object tokens。

Mechanisms 的视觉 token 消融结果却显示，真正决定定位的是物体区域内的视觉 token：移除 object-aligned tokens 会使定位性能大幅下降，而等量随机 token 的影响明显更小。进一步的 padding 和 shuffle 实验说明，模型会把物体相关视觉 token 看作一个定义空间范围的“容器”。

这意味着更符合 Mechanisms 结论的 FOCUS 目标可能是：

```text
query candidate visual tokens
↔ support bbox 内的 object visual tokens
```

而不是仅仅：

```text
query visual tokens
↔ support bbox 的坐标文本
```

建议分析轴为：

> **FOCUS 应该监督 query 对 BBOX 坐标文本的注意力，还是监督 query 与 support object visual tokens 之间的对应关系？**

建议设计以下控制实验：

* 保留 BBOX 坐标文本，但遮掉 support 框内物体；
* 保留 support 视觉目标，但删除坐标文本，改用视觉框覆盖；
* 在不同 support 图像之间交换 BBOX 坐标文本；
* 分别监督 query→BBOX-text 和 query→support-object-visual-token。

这些实验能判断 FOCUS 学到的是视觉对应关系、坐标读取捷径，还是二者结合。

---

### 4. FOCUS 是否应该显式拆分“实例识别”和“空间定位”？

FOCUS 去除了类别名称，目标对象完全通过 support 图像及其 bbox 指定。这样可以避免 category prior，但并不代表模型不再需要“识别”。模型仍然必须先判断 query 中哪个实例与 support 中的实例相同，再输出其空间范围。

Mechanisms 发现，分类和定位主要依赖不同的 attention heads，但消融 classification-critical heads 仍会显著损害定位。这被解释为一个顺序过程：

```text
object identification → spatial localization
```

即先形成对象身份表征，再由另一组更稀疏的 head 确定其空间范围。

因此，FOCUS 的单一 attention loss 可能混合了两个不同目标：

* 找到与 support 相匹配的实例；
* 确定该实例的精确边界。

建议将问题细化为：

> **FOCUS 的 attention loss 是否把 instance matching 与 spatial grounding 混在同一目标中？依据“先识别、后定位”的机制，是否应分别监督这两个阶段？**

一个更机制化的方案可以是：

```text
阶段 A：support–query instance identification
对 support 框内视觉 token 与 query 候选区域做对比学习；
加入同类别不同实例作为 hard negatives。

阶段 B：query spatial container formation
对已匹配实例的边界、范围和坐标进行定位监督。

阶段 C：GRPO
只负责优化最终坐标精度与输出格式。
```

这种解释也与 FOCUS 的消融现象相容：attention loss 主要解决“找对实例”，GRPO 进一步改善边界对齐。但这只是合理推断，FOCUS 原文没有证明两个训练阶段分别对应 Mechanisms 的识别与定位回路。

---

### 5. Containerization 对 FOCUS 的具体启示是什么？

Mechanisms 的核心发现是：物体相关 token 形成一个空间“容器”。人工将物体内部 token 复制到外围后，模型预测框也会随新的 token 范围扩大；而在物体区域内部打乱 token 排列，对 LLaVA 的定位影响很小。这说明定位更依赖“哪些空间位置包含物体相关信息”，而不完全依赖区域内部语义排列。

这对 FOCUS 的直接启示不是简单地“打乱 support 内部 token”，因为 ICOL 还必须依赖纹理、外观和局部结构区分同类别的不同实例。更合理的拆分是：

* **实例识别通路**应保留并利用内部语义和纹理；
* **定位通路**应重点学习物体相关 token 的空间范围和边界；
* 不应要求同一个表示同时对内部排列敏感又不敏感。

建议问题为：

> **能否将 FOCUS 中的 support 表示拆成 identity representation 与 spatial-container representation，使前者区分具体实例，后者预测对象范围？**

可加入两类机制约束：

* 身份分支：对内部 token shuffle 敏感；
* 定位分支：对轻微内部 shuffle 稳定，但对边界 dilation、erosion 和外围 distractor 敏感。

这比笼统地把所有 support 信息压入一个 attention loss 更符合 Mechanisms 的实验结论。

---

### 6. FOCUS 的 attention heatmap 是否足以证明模型真正使用了 visual support？

FOCUS 使用 answer token 的 attention heatmap 说明模型经过训练后更关注 query 图像，并将这作为视觉 grounding 改善的证据。

但 Mechanisms 没有把 attention 可视化直接当作因果解释，而是进一步使用：

* attention knockout；
* activation patching；
* causal mediation analysis；
* cumulative head ablation。

这些实验发现，可视化中存在的 attention 并不意味着相应 head 对结果有因果作用，真正的定位计算集中在少量 head 中。

因此建议加入一个方法论维度：

> **FOCUS 展示的 attention concentration 是性能提升的原因，还是训练后产生的相关现象？**

需要增加的证据包括：

* 屏蔽高 attention token 是否确实破坏定位；
* 屏蔽 FOCUS 强化的 head 是否消除增益；
* 将训练后关键 head activation patch 到基线模型，能否部分恢复 FOCUS 行为；
* 随机选择同等 attention mass 的 head 是否产生明显更小影响。

---

### 7. 能否通过稀疏 head 定向训练降低 FOCUS 的训练开销？

FOCUS 当前对 LoRA 参数进行两阶段训练：先使用 `LM loss + BBOX attention loss`，再使用 GRPO 优化 IoU 和格式奖励。

Mechanisms 表明定位计算具有低冗余性：少量关键 head 承载大部分因果作用，而大量低重要性 head 即使被删除，对定位影响也很小。作者因此明确提出 targeted head fine-tuning。

由此可以提出：

> **能否先在 Qwen2-VL 的 ICOL 输入上定位 support-conditioned critical heads，再仅对相关层或 head 应用 LoRA 与 attention loss？**

一个可验证的实验梯度是：

```text
全模型 LoRA
→ 仅机制关键层 LoRA
→ 仅关键 attention projection LoRA
→ 只训练 top-k causal heads
→ 无权重训练的 activation steering / patching
```

比较指标应包括 mIoU、训练显存、训练时长和跨数据集泛化。

这里必须保留一个重要限制：Mechanisms 只研究了 LLaVA-1.5 和 InternVL-3.5，且是单图、类别名称条件下的定位；FOCUS 主模型是 Qwen2-VL，并使用多图、无类别名称的实例定位。因此不能直接沿用论文发现的具体层号或 head 编号，必须在 Qwen2-VL 的 ICOL setting 中重新做因果定位。

## 当前最稳妥的综合判断

Mechanisms 对 FOCUS 的支持与质疑可以概括为：

1. **支持 FOCUS 的大方向**：FOCUS 消融表明只有加入 attention supervision 才产生主要性能提升；Mechanisms 则证明定位确实依赖稀疏、专门化、具有因果作用的 attention 通路。

2. **不完全支持 FOCUS 的具体 loss**：Mechanisms 强调 object-aligned visual tokens 和少量关键 head，但 FOCUS 监督的是所有层、所有 head 平均后的 query→BBOX-coordinate-text attention。

3. **提出了更强的改进方向**：将 FOCUS 从统一 attention 约束改为“实例识别—空间容器”两阶段机制，并在模型特定的 causal heads 上施加 support-object visual-token supervision。

4. **目前不能声称可免除 SFT**：Mechanisms 研究的是固定模型内部已有定位机制，没有证明只通过 prompt、activation patching 或 head steering 就能赋予模型可靠的 category-agnostic ICOL 能力。它支持“减少并定向训练”，但尚不支持“完全不训练即可替代 FOCUS”。

下一轮讨论最值得优先展开的是第 3 和第 4 个维度：**FOCUS 是否监督了错误的 token 类型，以及是否应将实例识别与空间定位拆开。**

* 屏蔽 FOCUS 强化的 head 是否消除增益；
* 将训练后关键 head activation patch 到基线模型，能否部分恢复 FOCUS 行为；
* 随机选择同等 attention mass 的 head 是否产生明显更小影响。

---

### 7. 能否通过稀疏 head 定向训练降低 FOCUS 的训练开销？

FOCUS 当前对 LoRA 参数进行两阶段训练：先使用 `LM loss + BBOX attention loss`，再使用 GRPO 优化 IoU 和格式奖励。

Mechanisms 表明定位计算具有低冗余性：少量关键 head 承载大部分因果作用，而大量低重要性 head 即使被删除，对定位影响也很小。作者因此明确提出 targeted head fine-tuning。

由此可以提出：

> **能否先在 Qwen2-VL 的 ICOL 输入上定位 support-conditioned critical heads，再仅对相关层或 head 应用 LoRA 与 attention loss？**

一个可验证的实验梯度是：

```text
全模型 LoRA
→ 仅机制关键层 LoRA
→ 仅关键 attention projection LoRA
→ 只训练 top-k causal heads
→ 无权重训练的 activation steering / patching
```

比较指标应包括 mIoU、训练显存、训练时长和跨数据集泛化。

这里必须保留一个重要限制：Mechanisms 只研究了 LLaVA-1.5 和 InternVL-3.5，且是单图、类别名称条件下的定位；FOCUS 主模型是 Qwen2-VL，并使用多图、无类别名称的实例定位。因此不能直接沿用论文发现的具体层号或 head 编号，必须在 Qwen2-VL 的 ICOL setting 中重新做因果定位。

## 当前最稳妥的综合判断

Mechanisms 对 FOCUS 的支持与质疑可以概括为：

1. **支持 FOCUS 的大方向**：FOCUS 消融表明只有加入 attention supervision 才产生主要性能提升；Mechanisms 则证明定位确实依赖稀疏、专门化、具有因果作用的 attention 通路。

2. **不完全支持 FOCUS 的具体 loss**：Mechanisms 强调 object-aligned visual tokens 和少量关键 head，但 FOCUS 监督的是所有层、所有 head 平均后的 query→BBOX-coordinate-text attention。

3. **提出了更强的改进方向**：将 FOCUS 从统一 attention 约束改为“实例识别—空间容器”两阶段机制，并在模型特定的 causal heads 上施加 support-object visual-token supervision。

4. **目前不能声称可免除 SFT**：Mechanisms 研究的是固定模型内部已有定位机制，没有证明只通过 prompt、activation patching 或 head steering 就能赋予模型可靠的 category-agnostic ICOL 能力。它支持“减少并定向训练”，但尚不支持“完全不训练即可替代 FOCUS”。

下一轮讨论最值得优先展开的是第 3 和第 4 个维度：**FOCUS 是否监督了错误的 token 类型，以及是否应将实例识别与空间定位拆开。**
