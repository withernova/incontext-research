# F-001 · 七个维度的核心，每条压缩到一两句：

## Finding

七个维度的核心，每条压缩到一两句：

1. **收益来自机制对齐而非普通训练**：FOCUS 消融显示只有加 attention loss 才大涨（36→68），普通 SFT/GRPO 反而掉点；Mechanisms 证明定位依赖稀疏因果 head——两者共同指向"FOCUS 有效是因间接干预了定位通路"。

2. **全层全头平均可能稀释因果通路**：FOCUS 对所有层头平均施加监督，但 Mechanisms 显示定位集中在极少数 head、部分 head 甚至是负贡献；建议只对 FOCUS 训练前后做 head knockout/CMA 验证真正命中了哪条通路。

3. **可能监督了错误 token**：FOCUS 监督的是 query→BBOX 坐标文本，而非 query→support 框内物体视觉 token；Mechanisms 显示决定定位的是物体区域视觉 token，更对口的监督应是后者。

4. **单一 loss 混了识别与定位**：Mechanisms 的"先识别后定位"顺序机制提示，FOCUS 的一个 attention loss 同时承担"找对实例"和"定准边界"两个目标，应拆成识别+定位两阶段监督。

5. **Containerization 启示双表示**：物体 token 形成"空间容器"，内部 shuffle 不伤定位但伤识别；建议把 support 拆成对 shuffle 敏感的身份表示 + 对边界敏感的定位表示，而非压进一个 loss。

6. **attention heatmap 不是因果证据**：FOCUS 用 attention 可视化当 grounding 证据，但 Mechanisms 用 knockout/patching/CMA 才确认因果；需补 head 屏蔽与 activation patching 验证。

7. **稀疏 head 训练可降本**：Mechanisms 提倡 targeted head fine-tuning；建议在 Qwen2-VL 的 ICOL 设定下重定位关键 head，再仅对其做 LoRA/attention loss，逐级比较 mIoU、显存、时长。注意不能直接套用 LLaVA/InternVL 的 head 编号。

一句话总结：**Mechanisms 支持 FOCUS 的干预大方向，但质疑其全层全头平均 + 监督坐标文本的具体设计，指向"稀疏 head + 物体视觉 token + 识别定位拆分"的更省、更因果的改进路线。**
