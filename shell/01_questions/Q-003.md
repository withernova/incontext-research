# Q-003 · IPLoc-ID 的跨图像 negative rejection 是否覆盖同图多实例的 reference–candidate identity binding？

## 原始疑问

IPLoc-ID 的F1能够衡量不同query image之间的positive/negative接受与拒绝，但是否充分衡量：当同一positive query image同时包含reference target与其他同类实例、而candidate bbox落到错误实例时，模型能否把candidate拒绝或纠正？如果F1只按query-level Yes/No计分，它是否高估了candidate-conditioned instance discrimination？

## 已核验事实

### 1. 论文定义与公开manifest的negative都是独立query image

论文明确规定每个sample包含共享reference的一个positive query和一个negative query，两者在训练/评估时作为独立query cases。negative从“different instance”采样：LaSOT/VastTrack来自不同sub-class/video sequence；PDM/GOT-10K来自不同class。[[personal2026]] §3.5.1–§3.5.3。

对公开仓库全部T2 manifests和构建脚本的审计结果：

```text
LaSOT train/test, N=1/2/4/8:
positive与inclass negative同路径=0
同sequence目录=0

VastTrack, N=1/2/4/8:
positive与inclass negative同路径=0
同subclass/sequence目录=0

GOT-10K, N=1/2/4/8:
positive与outclass negative同路径=0
同sequence目录=0

PDM, N=1/2:
positive与outclass negative同路径=0
同sequence目录=0
```

因此，公开POIL manifest没有显式构造“同一query image中同时含target A与distractor B，把B作为negative candidate”的counterfactual cell。它测试的是不同query image/sequence上的target presence或absence。

边界：tracking source的一张图可能偶然出现多个对象；单目标tracking annotation通常只标tracked instance。仅凭manifest不能证明每张图物理上绝无其他同类实例，也不能证明different sequence画面绝不偶然包含reference identity。但manifest没有为这些对象提供完整身份标注，也没有把candidate-level同图消歧定义成独立negative case。

### 2. 原文形式化声称candidate-conditioned verification，但标签仍是query-level

论文将生成分解为：

```text
p(B|x) · p(A|x,B,Q)
```

并称identification component验证generated BBOX是否对应reference instance。[[personal2026]] §3.3.1。

但ground-truth answer的定义是：

```text
Yes iff query image contains reference instance
No  iff query image does not contain reference instance
```

也就是`A*(x)`由query-level `δ(x)`决定。训练中的positive使用target GT box+Yes；negative使用另一图像中的plausible candidate box+No。[[personal2026]] §3.1.1, §3.3.2。

因此训练/评估没有显式提供这个关键冲突cell：

```text
同一positive图像确实包含reference target A
但给定candidate bbox指向同类non-target B
期望candidate-conditioned answer=No
```

这构成当前疑问的核心：形式上answer条件于B，但监督标签与F1未必识别“图像中有A、candidate却是B”的情况。

### 3. 原文有相关但不等价的limitation

作者明确写道：当前setting“focuses on a single object in each query image”，不处理多个target objects的同时定位和识别；future work将扩展到multi-object POIL。[[personal2026]] §5 Conclusion。

这是与本疑问最近的作者承认，但并不完全等价：

- 作者讨论的是同时定位/识别多个target objects；
- 本疑问即使最终只输出一个target，也要求在同图多个同类candidate间绑定正确identity；
- 原文没有明确讨论`Yes + wrong-instance bbox`、candidate-answer consistency、Joint F1@IoU、或identification-only F1对该失败的覆盖缺口。

论文还报告mIoU与F1并列，并把高F1解释为有效拒绝negative query images；正文未报告要求同一样本同时满足`Yes && IoU>=τ`的joint metric。[[personal2026]] §3.1.2, §4.1.2, §4.4。

## 当前判断

1. **可以确定**：公开manifest中的正式negative query并不是与positive相同的图像；LaSOT/VastTrack的in-class negative也来自不同sequence/subclass。
2. **不能宣称**：所有source images都绝对不存在同图其他实例。manifest/单目标tracking annotation不足以作这种全图实例排除。
3. **可以确定**：benchmark没有显式、可计分地构造“positive图像内错误同类candidate应被拒绝”的candidate counterfactual。
4. **作者部分触及**：承认single-object setting与multi-object limitation。
5. **作者未直接讨论**：query-level F1能否衡量candidate与reference的同图实例绑定，以及`Yes + wrong bbox`仍计identification TP的问题。

## 待检验假设

> IPLoc-ID 的F1主要测量query-level target-presence acceptance/rejection。它可能不能充分测量candidate-conditioned reference–instance binding，尤其是在positive图像同时存在target与同类distractor时。

该假设目前为`investigating`，不能写成已证实机制。R-004b只证实Joint F1 coverage gap；R-006只发现跨图像paired geometry假设不成立，并给出一些positive图像内wrong-instance的视觉线索。

## 判定实验

构造真实同图双实例四条件：

```text
reference A + query(A,B) + candidate A → 应Yes
reference A + query(A,B) + candidate B → 应No
reference B + query(A,B) + candidate A → 应No
reference B + query(A,B) + candidate B → 应Yes
```

要求candidate进入assistant autoregressive prefix，并以teacher-forced Yes/No logits判定。另对自然输出报告：

```text
target hit
wrong-instance hit
background/part/container
Joint F1@IoU
candidate-conditioned verifier accuracy
```

只有错误candidate仍得到高Yes margin，且这一现象在真实同图多实例数据上稳定出现，才能支持“verifier更接近query-level presence classifier而非candidate-bound verifier”。

## 公开manifest的双向reference审计（2026-07-21）

进一步逐文件检查全部公开T2 manifests，不仅检查positive/negative是否同图，还检查**同一query path是否在不同reference identity下重复出现**：

- LaSOT train/test，N=1/2/4/8：每个query path只出现一次，distinct-reference query reuse均为0；
- VastTrack test，N=1/2/4/8：同样为0；
- PDM N=1/2：部分query path重复，但reference set不变，distinct-reference query reuse为0；
- GOT-10K val N=1/2/4/8：各有3个query path被不同reference复用，但只是同一单目标tracking frame在一个record中作为自身positive、在另一无关类别record中作为out-class negative。例如同一stickleback frame相对stickleback reference为positive，相对convertible reference为out-class negative。这是跨类别query-level presence rejection，不是同图A/B candidate binding。

另外，LaSOT train manifest在**不同query images**之间存在28对reciprocal sequence-level negative edges（A-reference看B-sequence为negative，B-reference看A-sequence为negative）；它们仍不是同一张`query(A,B)`、没有candidate A/B双框，也不能组成目标四格。

因此可以在manifest层面确定：公开数据没有显式构造或计分以下双向同图四格：

```text
ref A + query(A,B) + candidate A
ref A + query(A,B) + candidate B
ref B + query(A,B) + candidate A
ref B + query(A,B) + candidate B
```

这支持“原benchmark/F1未显式覆盖candidate-level同图identity binding”的**评测覆盖判断**。它不证明模型一定失败，也不能证明原始source frame物理上没有额外实例。

## 已废弃的单向/背景设计：E003-R-010与R-011

R-010曾从R-006单人核验7个自然wrong-instance failures，并加入`target / wrong / matched background`。工程记录保留，但replay gate失败，不能回答本Q。经研究设计复核，背景条件对双向identity-binding主问题没有必要，后续实验已完全移除；历史产物仅为审计记录，不再作为证据或模板。

R-011试图修复same-load replay，但attempt-001在generation sequence trimming阶段失败，0 scientific records。该单向设计不再重试。下一正式run只采用真实`reference A/B × candidate A/B`四格，不设背景条件。

tokenization修正仍有效：prefix应结束于`?`，评分真实continuation token ` Yes`(7414)与` No`(2308)；旧R-005e的9454/2753 logits不可引用。
