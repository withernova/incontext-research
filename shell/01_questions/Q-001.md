# Q-001 · Mechanism 的 token 实验究竟在定位哪一层瓶颈？

## 原始疑问

论文 Mechanisms of Object Localization in Vision-Language Models 对 token 而非 patch 做实验，是否是因为作者认为 ViT 已经能够较好地提取视觉特征，而真正的问题是大模型缺乏像 ViT 一样良好使用这些特征的能力？

## 当前判断

### 从已有证据到机制问题

Mechanisms 论文的 token 实验并不是先验断言“ViT 已经完整解决视觉特征提取”，而是把干预放在 multimodal projection 后、LLM 输入前，以定位：**已经进入语言模型的视觉 token 如何被选择、路由并用于分类与定位**。论文发现 object-token ablation、attention knockout、head-level causal mediation 和 cumulative head ablation 能定位窄而稀疏的任务通路；分类与定位的关键 heads 只有有限重合，且定位依赖分类相关中间表示。[[mechanis2026]] §3.1 Visual Information Ablation；[[mechanis2026]] §3.3.1 Attention Knockout；[[mechanis2026]] §3.3.2 Causal Mediation Analysis；[[mechanis2026]] §3.3.3 Head Ablation Analysis

E-002/E-003 将问题推进到 personalized instance matching：

- E-002：bbox object-footprint 内部 token shuffle 的影响不稳定且有限，但 global visual-stream destruction 会使 negative rejection 崩溃；support/reference-like token contamination 会增加 same-class false positive。这更接近 `class-conditioned containerized reference matching`，尚不能证明稳定的 identity-detail binding。
- E-003：在本地 deterministic LaSOT reconstruction 上，identification F1=0.960，而 Joint F1@IoU0.5=0.764；大量回答 Yes 的 positive 框定位错误。这证明 metric-level decoupling，但没有说明内部原因。

因此，Q-001 当前最有价值的机制表述不是“attention 看了哪里”，而是：

> **对 Yes/No 判断有因果作用的 heads，是否读取与 reference identity 对应的正确 query-instance tokens；当 reference identity 或 candidate box 改变时，这些 heads 的读取区域和因果输出是否同步改变？**

### 核心机制假说

待证伪假说：当前 IPLoc-ID/Qwen3-VL 的正负决策可能由少量 identification heads 控制，但这些 heads 更依赖全局类别相似性、reference-like token 数量、显著 object container、类别文本或 bbox prefix，而没有稳定计算“reference identity 与指定 candidate region 是否匹配”。

这不等于“模型没看 query”。模型可能读取 query 全图、最显著同类物体或 unordered object-token aggregate，却仍未把 reference identity 绑定到 candidate 内视觉内容。

## 深究记录

### 1. 关键反事实：Reference A/B × Candidate A/B

应优先构造同一 query 中同时存在同类别实例 A、B 的四条件矩阵：

| Reference | Candidate | 正确答案 |
|---|---|---|
| A | box A | Yes |
| A | box B | No |
| B | box A | No |
| B | box B | Yes |

固定 query pixels、类别、背景、实例数量和 candidate objectness，只改变 reference–candidate identity relation。真正执行 instance verification 的 circuit 应随 reference A→B 把因果读取从 query-A 转到 query-B，并随 candidate A↔B 改变 Yes–No margin。

若真实双实例图像暂不可得，可先用 LaSOT 同类别不同 sequence 构造**标准化双面板/合成 query pilot**；该版本必须标为 synthetic causal-control pilot，矩形 crop/background leakage 是限制，不能替代真实多实例数据。

### 2. 先按因果效应选 heads，后看 attention

禁止先浏览所有 attention map 再挑“好看”的 head。推荐漏斗：

1. layer-level attention-output ablation；
2. top layers 内逐 head output ablation；
3. 对 top causal decision heads 做 activation patching / group-specific attention knockout；
4. cumulative top-head ablation 对比 equal-count low-effect heads；
5. 另找 bbox-coordinate heads，并比较两组 top-k 的 Jaccard overlap/rank correlation。

Decision score 使用 teacher-forced：

\[
M=\ell(\mathrm{Yes})-\ell(\mathrm{No})
\]

其中 `Yes`/`No` tokenization 必须现场校验。head ablation 应作用于每个 attention head 在 `o_proj` 混合前的输出 slice，而不是把整个 attention module 的混合输出误当单 head。

### 3. Token group

至少划分：

\[
S_{obj},S_{bg},Q_A,Q_B,Q_{bg},T_{category},T_{instruction},T_{bbox}.
\]

Qwen3-VL 必须记录 processor 后 `image_grid_thw`、merge 后 token grid、各图像 token span、bbox/mask→token overlap 和 selected-token counts。E-002 已知 pooler/deepstack 可能构成双路径，因此只分析 pooler token 不能宣称覆盖完整视觉信息流。

### 4. 三层指标及证据优先级

1. **Attention mass**：`A_h(G)=Σ_{j∈G} α_hj`，并报告按 token 数归一化版本；只回答“看了哪里”。
2. **Head-output/logit contribution**：按 Yes–No logit direction 分解，或用 group-level value/attention knockout 近似；回答读取内容推动 Yes 还是 No。
3. **Causal group effect**：`D_h(G)=M_clean-M_ablate(h,G)`；回答该 head 的决策作用是否依赖特定 token group。

证据优先级：causal group effect > output contribution > raw attention heatmap。

### 5. 预注册核心指标

- `Reference-conditioned routing shift`：reference A→B 时，关键 head 对 `Q_B-Q_A` 的 mass/contribution 差是否转正；
- `Target-over-distractor preference`：给定 reference A，`C_h(Q_A)-C_h(Q_B)`；reference B 时反向；
- `Candidate sensitivity`：`M(matched candidate)-M(distractor candidate)`；
- `Causal grounding selectivity`：target-group ablation 对 margin 的影响相对 distractor-group ablation；
- `Localization–identification head overlap`：bbox NLL heads 与 Yes/No margin heads 的 top-k Jaccard/rank correlation。

### 6. 可证伪结果模式

- decision heads 主要依赖文本/reference/global query：支持 image-level matching，而非 region verification；
- heads 看 query，但 reference A→B 不切换实例：支持 saliency/container 而非 identity binding；
- attention 切换但 causal contribution 不切换：瓶颈偏 value routing/readout；
- 中层正确切换、后层转向文本/global：支持 binding 在 autoregressive readout 中丢失；
- answer heads 正确绑定而 bbox heads错误：否定“弱 identity binding”主解释，问题更偏 spatial/coordinate readout。

## 必要性分析

这个实验是必要的，因为现有证据仍存在三种无法区分的解释：

1. 模型没有形成 reference-conditioned identity binding；
2. binding 已形成，但 bbox coordinate readout 失败；
3. bbox 候选错误，final verifier 又未检查 candidate region。

E-002 的 token shuffle/contamination 只表明输入证据敏感性，E-003 Joint F1/forced-candidate 只表明输出行为解耦；都不能定位 LLM 内哪条通路造成问题。四条件设计配合因果 head ablation 是从相关行为推进到 mechanism claim 的最短路径，并且能产生否定当前主假说的结果。

## 实现计划

### M1 · 数据与 prompt sanity

- 构造 2–4 个 synthetic double-instance samples；
- 验证 A/A 与 B/B margin 应高于 A/B 与 B/A；
- 若模型连该最小行为都不满足，只记录 failure，不进行大规模 head scan。

### M2 · Qwen3-VL head-ablation smoke

- 审计 decoder layer/head 数和 attention backend；
- 在 attention `o_proj` 输入处按 head slice 置零；
- 先扫 layer-level，再扫候选层内所有 heads；
- 保存 clean/ablated margin、delta、tokenization、hook module name 和完整性检查。

### M3 · Top-head causal routing

- top-20 decision heads；
- 对 `Q_A/Q_B/reference/background/text/bbox` 做 group-specific attention knockout 或 value contribution ablation；
- matched equal-token-count background/random controls；
- reference-switch routing shift。

### M4 · Localization circuit comparison

- teacher-force bbox coordinate tokens并以 coordinate NLL/perplexity 为 score；
- 重复 layer/head ranking；
- 计算 decision vs bbox top-k overlap；
- cumulative top-head vs low-effect-head ablation。

## 后续去向

- 新机制实验线建议记为 `E-004 · Reference-conditioned causal token routing`，避免把 head/circuit 实验混入 E-003 的 evaluation audit。
- E-003 保留 Joint F1、forced-candidate 和 low-IoU audit；E-004 承接四条件数据、causal head scan、token-group routing 与 localization/identification circuit overlap。
- 所有机制结论在 synthetic pilot 后仍需真实同图双实例和完整 deepstack-aware token mapping 复核。
